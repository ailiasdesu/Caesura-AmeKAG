# Caesura (AmeKAG) 引擎战略

> 更新: 2026-06-09 | 引擎完成度: ~80% (Alpha+)

## 核心目标

为视觉小说创作者提供**现代化、跨平台、AI 辅助开发**的 galgame 引擎。

## 定位

- **不是** KiriKiri 的克隆 — KAG 兼容是入口，不是终点
- **不是** 通用游戏引擎 — 聚焦视觉小说 + 轻量 3D 互动
- **是** 首个支持 AI IDE 辅助开发的 galgame 引擎

## 技术路线

### 已完成 (Alpha+)

1. **KAG 脚本兼容** — 61 个命令，覆盖文本/音频/图层/过渡/VFX/视频/存档
2. **D3D11 渲染管线** — 零 TDR，稳定 1-2ms GPU/帧
3. **跨平台架构** — SDL3 + bgfx + Lua 5.4，7 个纯虚接口
4. **资源管理** — 异步加载 + XP3/CARC 加密包 + 纹理预算
5. **多线程安全** — JobSystem (粒子/CARC/纹理)，主线顺序不阻塞
6. **Live2D Cubism 5** — Windows D3D11 就绪，条件编译，法律安全

### 当前阶段 (修复技术债)

2. Live2D 渲染路径补齐 (OpenGLShared + Metal)

### 中期目标

## 架构原则

- **AI 不嵌入引擎** — AI 在 IDE 中辅助开发，不在运行时
- **主线剧情单线程** — 保证流程确定性
- **多核用于辅助** — 资源加载/解析/解码/粒子/物理
- **?? Lua 层只走抽象接口 + BackendRegistry 工厂** — 禁止任何 Lua 脚本直接引用 C++ 具体类（如 BgfxRenderDevice）。所有 C++ 访问必须经过 `backend.lua → BackendRegistry → I*Backend 纯虚接口`。确保后端可替换、沙箱安全、AI 可理解。
- **Lua 沙箱安全** — require 白名单 + I/O 禁用 + 指令预算
