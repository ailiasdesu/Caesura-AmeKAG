# Caesura (AmeKAG) Strategy

## Target problem

会画画、会写剧本但完全不会编程的视觉小说创作者，情节和素材都准备好了，却无法用代码实现脑中想要的效果——现有引擎要么要求编程能力，要么无代码方案效果平庸，做不出专业质感。

## Our approach

押注 AI 辅助 KAG 脚本开发。AI 作为 IDE 中的编程搭档（类似 Codex），理解引擎 API 面，帮创作者把自然语言描述转为可运行的 KAG 脚本。引擎本身保持精简稳定，暴露清晰一致的 API 面供 AI 理解和使用。

## Who it's for

**Primary:** 艺术创作者——会画画、能写剧本、零编程基础。他们雇佣 AI + Caesura 一起来完成一件事：把准备好的情节和素材变成可运行的游戏，通过自然语言描述而不需要自己写代码。

## Key metrics

- **场景生成耗时** — 从创作者描述需求到场景可运行，目标 ≤ 30 分钟（通过一问一答反馈模式完成）
- **AI 代码一次通过率** — AI 生成的场景代码无需手动修改即可直接使用的比例，目标 90%
- **场景连续创作率** — 完成第一个场景后继续做第二个的比例
- **项目发布转化率** — 从新建项目到发布第一个可玩版本的比例

## Tracks

### 1. 引擎稳定化（API 面统一 + 安全可读）

从 18 条技术债中选出 4 条 AI 辅助开发的硬阻断项：合并 UnifiedBinding/KAGBinding 重复 API 面、ParticleSystem 切到抽象接口、简化 BackendRegistry 所有权模型、沙盒安全规则从 C++ 内联字符串外抽为独立 .lua 模块。

_Why it serves the approach:_ AI 需要一套清晰一致的 API 面才能生成正确代码——重复接口会让 AI 困惑，不稳定架构会让 AI 生成的脚本崩溃。

### 2. AI 辅助开发工具链

构建 IDE 侧 AI 辅助能力：KAG API 文档自动生成、AI 可读的沙盒规则文件、场景模板和代码片段库。让外部 AI（Codex/Copilot/Cursor）能理解引擎的能力边界和安全约束，生成正确、可运行的 KAG 脚本。

_Why it serves the approach:_ AI 作为 IDE 助手需要引擎侧的配合——清晰的 API 文档、可读的安全规则、可复用的模板——这些是 AI 生成质量的燃料。

### 3. Lua 沙盒极端安全

将现有沙盒提升到完整安全等级：AI 生成的任意 KAG 脚本必须无法造成引擎崩溃、内存泄漏或资源耗尽。隔离脚本执行上下文、限制单帧 CPU/内存预算、渲染操作白名单化。沙盒策略从"防御已知危险"升级到"默认拒绝，显式放行"。

_Why it serves the approach:_ AI 生成的代码不可信。当创作者说"我想要一个粒子效果"而 AI 生成了一段有问题的脚本，引擎不能崩——沙盒是创作自由的保障。
