## R5.1 — 触发完整 CI 运行

```
### 当前状态
- .github/workflows/ci.yml 有三端构建+测试
- macOS 的 || true 已移除
- Windows CI 已知绿（本地构建通过）
- macOS/Linux CI 从未在远端真实跑过

### Step 1: 首次触发
- 推送当前 master 到 GitHub
- 等待 GitHub Actions 完成（约 10-20 分钟）
- 观察三端结果

### Step 2: 根据失败日志修复（迭代）
- 查看失败 job 的日志
- 识别错误类型：
  a. 依赖安装失败 → 修复 apt/brew 命令或包名
  b. 构建配置失败 → 修复 CMake 平台判断
  c. 编译错误 → 修复平台相关代码（#ifdef, 类型差异等）
  d. 测试失败 → 修复测试或平台相关路径
- 每修复一个错误 → git commit → git push → 等待 CI
- 重复直到三端全绿

### Step 3: 最终确认
- 三端 CI badge 全部显示 "passing"
- README.md 中的 CI badge 链接正确
- 提交

### 常见预期错误及修复方向

#### macOS
- SDL3 找不到 → brew install sdl3 或设置 SDL3_DIR
- bgfx Metal 后端编译失败 → 检查 BX_PLATFORM_OSX 宏
- 测试找不到 SDL3.dylib → 设置 DYLD_LIBRARY_PATH

#### Linux
- SDL3 从源码构建失败 → 检查 apt 依赖（libx11-dev 等）
- 缺少 OpenSSL → apt install libssl-dev
- Wayland/X11 后端选择 → 确保安装了 libwayland-dev

#### Windows (应该已经绿)
- 如果失败 → 检查 vcpkg SDL3 安装

### 预期产出
- 三端 CI 全部绿色
- N 次提交（取决于错误数量）
```
