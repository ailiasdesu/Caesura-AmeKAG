## F1 — CI 三端验证

```
推送 master 到 GitHub → 观察 Actions → 根据失败日志修复 → 三端全绿。

已知：macOS 曾用 || true 跳过（已移除），Linux 未远端验证过。

Step 1: 推送触发
git push origin master

Step 2: 观察 GitHub Actions 结果
- Windows: 预期绿（本地 MSVC 构建通过）
- macOS: 可能因 brew 依赖或 Metal 后端失败
- Linux: 可能因 SDL3 源码构建或依赖缺失失败

Step 3: 修复迭代
macOS 常见修复：
- sdl3/freetype brew 安装 → 设置 CMAKE_PREFIX_PATH
- bgfx Metal 后端宏 → BX_PLATFORM_OSX 检查

Linux 常见修复：
- SDL3 源码构建 → 补齐 apt 依赖（libx11-dev libxrandr-dev 等）
- 测试 CWD → 确保从 build/tests 目录运行
- 耦合计数脚本 → python3 可用性

每修复一个 → 提交 → 推送 → 等待 CI

Step 4: 三端全绿后
- README.md 更新 CI badge
- 提交

预期：N 次提交，最终三端 CI 全绿。
```
