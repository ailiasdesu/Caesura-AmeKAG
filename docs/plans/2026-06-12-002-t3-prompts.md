## T3.1 — CMakeLists.txt 跨平台兼容

```
### 任务：修复 CMakeLists.txt 跨平台问题

### 当前已知问题
- SDL3_DIR 硬编码 Windows 路径
- bgfx 编译选项按平台分支不完整
- Live2D Cubism SDK 路径假设 Windows D3D11
- 部分 #include 使用反斜杠路径
- 测试资源复制命令 Windows-only

### 执行

#### Step 1: SDL3 跨平台查找
- 读取 CMakeLists.txt 中的 SDL3 find_package
- 修改为：
  - Windows: 优先使用 external/SDL3 预编译包
  - Linux: find_package(SDL3) 或 pkg-config
  - macOS: find_package(SDL3) 或 Homebrew 路径
- 使用 CMAKE_SYSTEM_NAME 判断平台
- 提交

#### Step 2: bgfx 平台后端选择
- 检查 bgfx 编译配置
- Windows: D3D11（默认）
- Linux: OpenGL
- macOS: Metal
- 确认 CMake 正确传递后端宏
- 提交

#### Step 3: 修复路径和测试资源
- 扫描所有 CMakeLists.txt 中的反斜杠路径（\\ → /）
- 测试资源复制命令改为跨平台（cmake -E copy）
- 提交

### 预期产出
- CMakeLists.txt 三端可配置
- 3 次提交
```

## T3.2 — Linux CI

```
### 任务：添加 Linux CI job

### 执行

#### Step 1: Linux CI 配置
- 编辑 .github/workflows/ci.yml
- 添加 job:
  ```yaml
  build-linux:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install deps
        run: |
          sudo apt-get update
          sudo apt-get install -y libsdl3-dev libfreetype-dev
      - name: Configure
        run: cmake -B build -DCMAKE_BUILD_TYPE=Debug
      - name: Build
        run: cmake --build build --config Debug --parallel
      - name: Test
        run: cd build && ctest --output-on-failure -C Debug
  ```
- 提交并推送，观察 CI 结果

#### Step 2: 修复 Linux 构建错误
- 根据 CI 日志逐项修复
- 常见的：平台宏差异、include 路径大小写、类型转换
- 每次修复一个错误 → 提交 → 推送观察
- 直到 Linux CI 变绿

### 预期产出
- Linux CI job 绿
- N 次提交（取决于错误数量）
```

## T3.3 — macOS CI

```
### 任务：添加 macOS CI job

### 执行

#### Step 1: macOS CI 配置
- 编辑 .github/workflows/ci.yml
- 添加 job（类似 Linux，但用 brew 安装依赖）：
  ```yaml
  build-macos:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install deps
        run: brew install sdl3 freetype
      - name: Configure
        run: cmake -B build -DCMAKE_BUILD_TYPE=Debug
      - name: Build
        run: cmake --build build --config Debug --parallel
      - name: Test
        run: cd build && ctest --output-on-failure -C Debug
  ```
- 提交并推送，观察 CI 结果

#### Step 2: 修复 macOS 构建错误
- Metal 后端可能需要额外配置
- 根据 CI 日志逐项修复
- 直到 macOS CI 变绿

### 预期产出
- macOS CI job 绿
- N 次提交
