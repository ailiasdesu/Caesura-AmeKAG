## R1 — 编辑器集成

```
### R1.1: 验证 web-editor ↔ Engine RPC 通信链路

当前状态：web-editor/ 是 React+Vite 前端，Engine 在 editorMode 时启动 EditorServer (HTTP :9876)。链路从未端到端验证过。

Step 1: 启动 Engine 编辑器模式
- 修改 main.cpp 或 EngineConfig，设置 editorMode=true
- 启动引擎，确认 EditorServer 在 9876 端口监听
- 确认日志输出 "[EditorServer] started on port 9876"

Step 2: 验证 RPC 端点
- 用 curl 测试 EditorServer 的 HTTP API：
  curl http://localhost:9876/api/status
  curl http://localhost:9876/api/scenes
- 如果端点不可用，检查 EditorServer.cpp 中的路由注册

Step 3: 验证编辑器前端连接
- cd web-editor && npm install && npm run dev
- 打开浏览器 → 确认 StageView 显示连接状态
- 在 CodeEditor 中输入 KAG 脚本 → 点击运行 → 确认 Engine 收到脚本
- 提交

预期产出：编辑器与引擎的 RPC 通信链路端到端可用。

---

### R1.2: Live2D 预览面板

当前状态：Live2DPanel.jsx 是占位符。StageView 可渲染 bgfx 帧。

Step 1: 查阅 Live2DPanel.jsx 和 StageView.jsx
- 理解当前的占位符结构和 StageView 渲染方式

Step 2: 实现模型加载流程
- Live2DPanel 添加 "Load Model" 按钮
- 通过 RPC 调用 Engine 加载 .moc 文件
- 确认 Engine 端 Live2DBackend（或 NullAnimationBackend）响应请求
- 提交

预期产出：Live2D 面板可触发模型加载请求。

---

### R1.3: 一键打包

当前状态：tools/carc_pack 是独立 CLI 工具。CARCWriter 已实现。

Step 1: 在 EditorServer 中添加打包端点
- POST /api/build → 调用 CARCWriter 打包当前项目
- 返回 .carc 文件路径

Step 2: 前端 Build 按钮
- AssetPanel.jsx 添加 "Build" 按钮
- 调用 /api/build → 显示打包结果（成功/失败/文件路径）
- 提交

预期产出：创作者可在编辑器中点击按钮打包项目。
```
