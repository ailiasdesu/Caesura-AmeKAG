## R2.1 — 无 SDK 环境下降级方案

```
### 当前状态
- NullAnimationBackend 所有方法返回安全默认值（-1, false, nullptr）
- loadModel 返回 -1（模型未加载）
- update/render 为空操作
- Demo 中无法展示任何立绘

### Step 1: PNG 立绘降级实现
- 修改 src/live2d/NullAnimationBackend.h/.cpp：
  - 添加私有成员：std::unordered_map<int, bgfx::TextureHandle> m_staticTextures
  - loadModel(const char* path) 重写：
    - 如果 path 以 .png/.jpg/.bmp 结尾 → 调用 TextureManager 加载为静态纹理
    - 存储到 m_staticTextures，返回有效 handle
    - 其他格式返回 -1（保持原有行为）
  - update(int modelId, float dt) 重写：
    - 如果 modelId 在 m_staticTextures 中 → 调用 render 绘制静态纹理
  - render(int modelId) 重写：
    - 使用 BgfxDraw 或直接调用 bgfx 提交静态纹理到当前视图
  - destroyModel(int modelId)：清理纹理
- 注意：NullAnimationBackend 需要访问 TextureManager 和 IRenderDevice
  - 通过 BackendRegistry::instance().getTextureManager() 和 getRenderDevice()
- 提交

### Step 2: Demo 立绘测试
- 在 tests/scripts/ 下放一个 64x64 测试 PNG（或从现有测试资源复用）
- 扩展 tests/cpp/test_live2d.cpp：
  - 测试用例：loadModel("test.png") 返回有效句柄
  - 测试用例：update/render 不崩溃
  - 测试用例：destroyModel 后纹理释放
- 构建 + 测试全绿后提交

### 预期产出
- 无 Cubism SDK 时 Demo 可展示静态 PNG 立绘
- 2 次提交
```

## R2.2 — Cubism SDK 集成文档

```
### Step 1: 撰写集成文档
- 创建 docs/guides/live2d-setup.md
- 内容结构：

# Live2D Cubism SDK 集成指南

## 前置条件
- Live2D Cubism SDK for Native 许可（免费获取：https://www.live2d.com/download/cubism-sdk/）
- 支持平台：Windows (D3D11) / macOS (Metal) / Linux (OpenGL)

## 安装步骤

### 1. 下载 SDK
从 Live2D 官网下载 Cubism SDK for Native R5 或更高版本。

### 2. 放置 SDK 文件
将 SDK 解压到 external/Live2D/ 目录下，确保结构为：
external/Live2D/
  Core/
  CubismNativeFramework/
  Framework/

### 3. CMake 配置
cmake -B build -DCAESURA_ENABLE_LIVE2D=ON
cmake --build build --config Debug

### 4. 验证
构建成功后，Demo 中的 Live2D 模型将使用真实 SDK 渲染（动态形变/眨眼/口型同步）。

## 常见问题
- Q: 构建提示找不到 CubismNativeFramework
  A: 确认 SDK 文件放置在正确的目录结构中
- Q: 运行时崩溃
  A: 确认 .moc 模型文件路径正确，且 .moc3 格式兼容
- Q: macOS 上 Metal 渲染路径不可用
  A: 自动回退到 OpenGL 渲染路径
```

- 提交
