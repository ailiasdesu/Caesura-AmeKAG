## 缺口

Engine 构造函数用三元表达式创建 GpuMonitor/NullGpuMonitor，直接 include 两个具体头文件。测试中构造 Engine 仍需 NullGpuMonitor.h。

当前代码（src/entry/Engine.cpp:70）：
```cpp
m_gpuMonitor(config.gpuMonitor 
    ? std::unique_ptr<IGpuMonitor>(config.gpuMonitor) 
    : (m_config.headless 
        ? std::unique_ptr<IGpuMonitor>(std::make_unique<NullGpuMonitor>()) 
        : std::unique_ptr<IGpuMonitor>(std::make_unique<GpuMonitor>())))
```

## Step 1: 重构构造函数

```
目标：Engine 构造函数不再直接创建或 include GpuMonitor/NullGpuMonitor。

执行：
- Engine() 初始化列表中 m_gpuMonitor 改为 nullptr（或删除该行）
- 移除 Engine.cpp 顶部的 #include "GpuMonitor.h" 和 #include "NullGpuMonitor.h"
- 在 initPlatformPhase() 中，FreeTypeContext init 之前添加：
  if (m_config.headless)
      m_gpuMonitor = std::make_unique<NullGpuMonitor>();
  else
      m_gpuMonitor = std::make_unique<GpuMonitor>();
- 这 2 个 include 移到 Engine.cpp 中（仅在 initPlatformPhase 需要）
- 构建 + 测试全绿后提交
```
