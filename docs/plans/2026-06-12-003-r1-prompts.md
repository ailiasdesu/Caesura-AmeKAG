## R1.1 — 验证 web-editor ↔ Engine RPC 通信

```
### 当前状态
- web-editor/ 有完整的 React 前端（StageView, CodeEditor, AssetPanel 等）
- Engine 在 editorMode=true 时启动 EditorServer (HTTP :9876)
- src/rpc/EditorServer.cpp 注册了 /api/* 路由
- 这个链路从未端到端验证过

### Step 1: 确认引擎端 RPC 可用
- 检查 EditorServer.cpp 中注册的路由：
  rg 'handle_' src/rpc/EditorServer.cpp
- 确认至少有以下端点：
  GET  /api/status
  GET  /api/scenes
  POST /api/execute
  POST /api/build
- 如果缺失，添加基本端点
- 添加测试：tests/cpp/test_rpc.cpp 中加入端点注册测试
- 构建 + 测试全绿后提交

### Step 2: 验证 HTTP 通信
- 修改 main.cpp 或 EngineConfig 使引擎以 editorMode=true 启动
- 在 headless 模式下启动引擎 → 用 curl 测试端点（或写代码测试）
- 确认 GET /api/status 返回 JSON {"status":"ok"}
- 提交

### Step 3: 验证前端连接
- 读取 web-editor/src/context/EditorContext.jsx 中的 API 调用
- 确认 API base URL 指向 http://localhost:9876
- 确认 StageView.jsx 能通过 RPC 获取引擎状态
- 启动引擎 + npm run dev → 浏览器验证连接
- 提交

### 预期产出
- editor ↔ engine RPC 链路可用的端到端验证
- 3 次提交
```
