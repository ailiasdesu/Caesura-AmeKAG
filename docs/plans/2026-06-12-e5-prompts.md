## 缺口

BgfxMiniGameBackend 全部 no-op。有结构定义（MiniGeometry/MiniCollision/MiniMaterial/MiniLight）和 shader（EmbeddedShaders_MiniGame），无实现。

**目标（60%）：** 最小可用的 3D 场景渲染链路。

**不做：** 物理引擎、骨骼动画、光照阴影、碰撞响应——这些是完整 3D 引擎的工作量。

## Step 1: GLB 场景加载

```
文件：src/minigame/BgfxMiniGameBackend.cpp

修改 loadScene(const std::string& path)：

1. 使用 cgltf 解析 GLB 文件
   - cgltf.h 在 external/bgfx/bgfx/3rdparty/cgltf/cgltf.h
   - cgltf_parse_file() + cgltf_load_buffers()

2. 提取第一个 mesh 的顶点数据
   - 遍历 cgltf_data.meshes[0].primitives[0]
   - 读取 POSITION 属性的顶点位置（float*3）
   - 读取索引数据（如有）

3. 创建 bgfx 资源
   - bgfx::VertexBufferHandle（用 bgfx::createVertexBuffer）
   - bgfx::IndexBufferHandle（如有索引）
   - bgfx::VertexLayout（POSITION + NORMAL + TEXCOORD0）

4. 存储到内部结构
   struct SceneData {
       bgfx::VertexBufferHandle vbh;
       bgfx::IndexBufferHandle ibh;
       uint32_t vertexCount;
       uint32_t indexCount;
   };
   std::unordered_map<uint32_t, SceneData> m_scenes;

5. 无效文件返回 0，有效返回 handle

构建 + 测试全绿后提交。
```

## Step 2: 场景渲染

```
修改 render()：

1. 获取当前 active 场景的 SceneData
2. 如果无 active 场景 → 直接返回
3. 设置视图：
   - bgfx::setViewRect(MINIGAME_VIEW_ID, 0, 0, width, height)
   - bgfx::setViewClear 清屏
4. 绑定资源：
   - bgfx::setVertexBuffer(0, sceneData.vbh)
   - bgfx::setIndexBuffer(sceneData.ibh)
5. 设置变换矩阵（单位矩阵或简单旋转）
6. 使用 EmbeddedShaders_MiniGame 的 program
7. bgfx::submit(MINIGAME_VIEW_ID, program)

关键约束：
- MINIGAME_VIEW_ID 需要在 BgfxRenderDevice 中已预留
  检查 BgfxDeviceCore::setupDefaultViews()
- 渲染状态：BGFX_STATE_DEFAULT | BGFX_STATE_CULL_CW

构建 + 测试全绿后提交。
```

## Step 3: 场景卸载与清理

```
修改 unloadScene(uint32_t handle) + shutdown()：

unloadScene:
- 从 m_scenes 查找 handle
- bgfx::destroy(m_scenes[handle].vbh)
- bgfx::destroy(m_scenes[handle].ibh)
- m_scenes.erase(handle)

shutdown:
- 遍历 m_scenes 销毁所有 bgfx 资源
- m_scenes.clear()
- 调用父类 shutdown 或设置 m_initialized = false

构建 + 测试全绿后提交。
```

3. "MiniGame: two loadScene calls return different handles"
