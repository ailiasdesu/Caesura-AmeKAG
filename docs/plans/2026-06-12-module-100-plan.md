# 模块 100% 完善计划

> 创建日期: 2026-06-12 | 状态: active | 基线: 306/306

## 诚实的前言

"100%"对不同模块含义不同。entry 的 90%→100% 是修一个构造函数问题。MiniGame 的 30%→100% 是实现完整 3D 渲染管线——这是两个数量级的任务。

render 模块的 bgfx 类型暴露在接口中——这不是 bug，是设计取舍。AGENTS.md 已标记为 deferred，本次不纳入。

## 目标模块

| 模块 | 当前 | 目标 | 性质 |
|------|------|------|------|
| **entry** | 90% | 100% | 修复构造函数脆弱性 |
| **rpc** | 95% | 100% | 确认前端面板完整性 |
| **script** | 90% | 100% | 补齐边界测试 |
| **storage** | 95% | 100% | 实现缩略图截图 |
| **minigame** | 30% | 60% | 完成 BgfxMiniGameBackend 核心渲染链路 |
| **render** | 95% | 95% | bgfx 类型 deferred，不纳入 |

## 赛道

### E1: entry 模块 — 构造函数解耦

**当前缺口:** Engine 构造函数用三元表达式选择 GpuMonitor 或 NullGpuMonitor，直接 include 两个具体头文件。测试中构造 Engine 仍需 NullGpuMonitor 头文件。

**目标:** Engine 构造函数不再 include GpuMonitor.h 或 NullGpuMonitor.h。GpuMonitor 创建推迟到 `initPlatformPhase()`。

**IU E1.1:** 重构 Engine 构造函数
- `m_gpuMonitor` 默认 nullptr
- `initPlatformPhase()` 中创建 GpuMonitor（或 NullGpuMonitor for headless）
- 删除 `#include "GpuMonitor.h"` 和 `#include "NullGpuMonitor.h"` 从 Engine.cpp
- 添加 `#include "NullGpuMonitor.h"` 到 initPlatformPhase 或测试文件

**IU E1.2:** 扩展 test_entry.cpp
- Engine 默认构造不崩溃（无 NullGpuMonitor 依赖）
- Engine headless init → shutdown 完整生命周期

**验收:** Engine.cpp 不再 include 任何 GpuMonitor 头文件。306+ 测试全绿。

### E5: minigame 模块 — 核心渲染链路

**当前缺口:** BgfxMiniGameBackend 的 loadScene/enter/leave/render 全部是 no-op。但有 MiniGeometry、MiniCollision、MiniMaterial、MiniLight 结构定义和 EmbeddedShaders_MiniGame。

**目标（60%）:** 完成最小可用的 3D 场景渲染——加载 GLB → 渲染到视图 → 卸载。

**不做（超出范围）:**
- 真实物理引擎
- 骨骼动画
- 光照/阴影系统
- 完整碰撞响应

**IU E5.1:** 场景加载
- `loadScene(path)` 使用 cgltf 解析 GLB 文件
- 提取 mesh 数据（顶点/索引）
- 创建 bgfx vertex/index buffer

**IU E5.2:** 场景渲染
- `render()` 提交所有 mesh 到 MINIGAME 视图
- 使用 EmbeddedShaders_MiniGame

**IU E5.3:** 场景卸载
- `unloadScene()` 释放所有 bgfx 资源
- `shutdown()` 确保全部清理

**IU E5.4:** 测试
- test_minigame.cpp：
  - loadScene 不存在的 GLB 返回 0
  - loadScene 后 enter → render → leave 不崩溃
  - shutdown 后资源清零

**验收:** BgfxMiniGameBackend 可加载渲染简单 3D 场景。306+ 测试全绿。

```
E1 (entry) → E4 (storage) → E3 (script) → E2 (rpc) → E5 (minigame)
```

E1 先做（最小改动，最大效果）。E4/E3 可并行。E2 快速审查。E5 最后（工作量大）。
