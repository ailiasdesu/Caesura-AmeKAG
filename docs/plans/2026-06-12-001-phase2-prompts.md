## M4 — archive 模块完善

```
## 任务：archive 模块完善（M4）

### 当前状态
- 6 cpp + 7 h，实现了 CARC 格式的加密/压缩/读写
- 无 api/ 目录，无测试文件
- 关键类：CARCReader, CARCWriter, CryptoEngine, CarcAssetProvider, CRLManager, DeltaCARC
- CryptoEngine 是 namespace Caesura::carc 下的静态方法类
- storage/SaveManager.cpp 直接 include archive/CryptoEngine.h
- BackendRegistry 中未注册任何 archive 接口

### 执行步骤

#### Step 1: 提取 IArchiveReader / IArchiveWriter 接口
- 读取 CARCReader.h 和 CARCWriter.h 的公共 API
- 创建 src/archive/api/IArchiveReader.h（纯虚：open, read, close, getFileCount 等）
- 创建 src/archive/api/IArchiveWriter.h（纯虚：create, addFile, finalize 等）
- CARCReader 继承 IArchiveReader，CARCWriter 继承 IArchiveWriter
- 提交

#### Step 2: 提取 ICryptoEngine 接口
- 创建 src/archive/api/ICryptoEngine.h
- 将 CryptoEngine 的公共方法声明为纯虚：
  encrypt/decrypt/generateKey/generateNonce/generateKeyPair
  sign/verify/sha256
  readPublicKey/readPrivateKey/writePublicKey/writePrivateKey
- 参数中的固定大小数组改为指针+长度（接口不应暴露 CARCFormat.h 中的 AES_KEY_SIZE 宏）：
  const uint8_t key[32] → const uint8_t* key, size_t keyLen
- CryptoEngine 改为实例类，实现 ICryptoEngine
- 注意：CryptoEngine 是静态方法类，改为实例类需要修改所有调用方
  - 搜索 CryptoEngine:: 调用（SaveManager.cpp, CARCReader.cpp 等）
  - 改为通过 BackendRegistry 获取 ICryptoEngine*
- 提交

#### Step 3: CarcAssetProvider 注册到 BackendRegistry
- 检查 CarcAssetProvider 是否已实现 IAssetProvider
- 在 BackendRegistry 中添加 setArchiveReader/getArchiveReader
- Engine::init() 中注册
- 提交

#### Step 4: 添加测试
- 创建 tests/cpp/test_archive.cpp
- 测试用例（至少 4 个）：
  1. CARCReader 打开无效文件返回 nullptr/false
  2. CARCWriter 创建/写入/读取往返
  3. ICryptoEngine 加解密往返（生成随机 key → 加密 → 解密 → 验证明文一致）
  4. CarcAssetProvider 通过 IAssetProvider 接口查询不存在的文件
- 确保测试不依赖特定路径（使用临时目录）
- 构建 + 测试全绿后提交

### 预期产出
- src/archive/api/IArchiveReader.h
- src/archive/api/IArchiveWriter.h
- src/archive/api/ICryptoEngine.h
- tests/cpp/test_archive.cpp
- 4 次提交
```

## 模块间依赖

```
M4 (archive) ──→ M5 (storage)  [SaveManager 需要 ICryptoEngine]
M6 (script)  ──→ 独立，可并行
```

如果 M4 尚未完成，M5 Step 2（解除 archive 依赖）会需要 ICryptoEngine 接口存在于 BackendRegistry 中。可以：
- A) 先完成 M4，再做 M5
- B) M5 跳过 Step 2，先做 Step 1/3/4，Step 2 留待 M4 完成后补
