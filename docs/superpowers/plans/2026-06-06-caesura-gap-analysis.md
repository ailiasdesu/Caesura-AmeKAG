## Summary

基于说明书 66 项决策逐项审计当前 C++ (41 .cpp / 46 .h) + Lua (42 .lua) 代码库，产出差距清单并按优先级分派到 G7-G10 四个阶段。目标：G7 修复阻塞性渲染问题 → G8 视觉验证 + 着色器补全 → G9 P2 高价值功能 → G10 生产就绪。

## Gap Analysis

### 1. 🔴 阻塞性问题（G7 必须解决）

| ID | 问题                                                         | 位置                                                                       | 影响                                                     |
| :- | :--------------------------------------------------------- | :----------------------------------------------------------------------- | :----------------------------------------------------- |
| B1 | **ShaderCache::compileVariant() 返回 BGFX\_INVALID\_HANDLE** | `src/Render/ShaderCache.cpp:94`                                          | 所有 blend/transition/vfx 着色器在运行时不可用                     |

| B3 | **引擎从未以图形模式启动验证**                                          | 全局                                                                       | 首帧渲染、图层合成、输入事件未端到端验证                                   |
| B4 | **FontRenderer 实际渲染未验证**                                   | `src/Render/FontRenderer.cpp`                                            | 文本显示链（FreeType→TextureArray→bgfx submit）未测试            |

## Phase Plan

### G7: 阻塞修复 + 首帧渲染 🔴

**目标**: 修复着色器 → 启动引擎 → 看到画面

| Unit  | 工作                                                | 验证标准                 |
| :---- | :------------------------------------------------ | :------------------- |
| G7-U1 | 修复 ShaderCache::compileVariant 返回有效 ProgramHandle | bgfx 无着色器错误          |
| G7-U2 | 重新编译 blend/transition/vfx 的 DXBC/SPIR-V 字节码       | EmbeddedShaders 全部合法 |
| G7-U3 | 端到端启动测试：SDL窗口→bgfx初始化→清屏→提交三角形                    | 看到 bgfx 清屏颜色         |
| G7-U4 | 单图层纹理渲染验证：加载 PNG → createTexture → submit         | 窗口显示纹理               |
| G7-U5 | 文本渲染验证：FreeType 初始化 → 渲染中文 → 显示                   | 窗口显示中文               |
| G7-U6 | bgfx 故障时 SDL\_ShowSimpleMessageBox 触发             | 无 GPU 驱动时不黑屏         |

**产出**: 可运行的图形引擎（基础渲染链验证通过）

### G8: 视觉验证 + 着色器补全 🟡

## Execution Order

```
G7  ──→ G8  ──→ G9  ──→ G10
(阻塞)  (验证)  (扩展)  (生产)
```

## Assumptions

* G7 着色器修复使用现有 shaderc 工具链，不引入新编译依赖

* G8 不需要完整美术素材（使用占位纹理验证渲染管线）

* G9 3D 仅需 `bgfx::createUniform + cube vertices`，不引入完整 3D 引擎

* G10 移动平台适配仅桌面模拟验证（真实设备测试为上线前事项）

* 所有阶段使用 `@superpowers:subagent-driven-development` 模式执行
