## Constraints

- **Lua 5.4 allocator rule:** Never call lua_gc(LUA_GCCOLLECT) inside a lua_Alloc hook.
  Use a flag + main-loop check pattern instead.
- **C function delegation rule:** C binding functions must not delegate to Lua-side functions
  of the same name. If delegation is needed, use a different function name to break cycles.
- **Sandbox globals:** Any lua_setglobal call in Engine.cpp must have a corresponding entry
  in sandbox.lua's _G_whitelist.
- **Texture lifecycle:** Avoid per-frame create/destroy cycles. Throttle animated textures

## Build System & CI

### Cross-Platform Guard Pattern
All Windows-specific headers and APIs must use `#ifdef _WIN32` guards. The non-standard
`CAESURA_PLATFORM_WINDOWS` macro is deprecated 鈥?use standard `_WIN32` instead.
MSVC-specific CRT debug APIs (`_CrtSetReportHook`, `_CrtSetReportMode`) require
`#ifdef _MSC_VER` guards covering both includes and function bodies.

### BX_CONFIG_DEBUG
bgfx debug configuration macro. Must be defined for **all platforms**, not only MSVC.
Use CMake generator expressions: `$<$<CONFIG:Debug>:BX_CONFIG_DEBUG=1>$<$<NOT:$<CONFIG:Debug>>:BX_CONFIG_DEBUG=0>`
in a global `target_compile_definitions`, outside any platform-conditional block.

### Foundation Framework (macOS Metal Backend)
bgfx Metal renderer requires Apple Foundation framework for Objective-C runtime support
(`NSAutoreleasePool`, `objc_msgSend`). Both main engine and test targets must link:
`find_library(FOUNDATION_LIBRARY Foundation REQUIRED)`.

### doctest Array Compatibility
When passing `extern const T arr[]` to doctest `CHECK(arr != nullptr)`, macOS Clang
(libc++) fails because `operator<<` doesn't accept array types. Fix:
`CHECK(static_cast<const void*>(arr) != nullptr)`.

### GitHub Actions CI
Multi-platform CI (Windows MSVC, macOS Clang, Linux GCC) with SDL3 via vcpkg.
Windows tests marked `continue-on-error: true` because bgfx D3D11 requires GPU
not available on headless runners. See docs/solutions/build-error/cross-platform-ci-fixes.md.
